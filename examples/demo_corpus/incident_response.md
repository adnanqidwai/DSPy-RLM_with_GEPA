# Incident Response

The first incident response record should name the owner, preserve the timeline, list the
mitigation in progress, and capture the rollback decision.

A useful postmortem separates customer impact, detection signals, root cause, and follow-up
actions so that responders do not confuse symptoms with long-term prevention work.

Auditors connect dashboards, alerts, runbooks, and escalation records to understand whether the
response procedure matched what monitoring showed during the incident.
