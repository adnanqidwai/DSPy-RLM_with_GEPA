# Software Delivery

A canary deployment compares baseline behavior, error rate, rollback criteria, and the exposed
cohort before the rollout expands to more traffic.

Dependency risk reviews use an SBOM, provenance records, vulnerability entries, and patch status to
show where a risky package came from and whether it has been remediated.

Observability connects traces, spans, latency, and saturation so engineers can diagnose whether a
slow request is caused by service code or capacity pressure.
